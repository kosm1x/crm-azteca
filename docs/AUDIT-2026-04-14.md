# Full System Audit ŌĆö 2026-04-14

## Scope

6 parallel specialist audit agents + manual verification covered:
speed, resilience, logic/correctness, memory/context, intelligence/ACI,
and security. Each finding was verified by direct file read before being
accepted, and false positives were explicitly excluded. 71 raw findings
collected ŌåÆ 58 verified real ŌåÆ all fixed in this session. 13 rejected
or downgraded.

All 1119 tests pass after the fixes (no regressions). No new tables, no
schema migrations beyond an additive composite index. Pilot bundle
(phases 8 + 9 + 11 + Hindsight + approvals) remains feature-complete.

## What landed

### Critical correctness

- **Hierarchy queries now filter inactive staff.** `isManagerOf` and
  `isDirectorOf` in `crm/src/hierarchy.ts` both previously lacked
  `AND activo = 1`, so approval workflows could route through deleted
  managers/directors. Fixed in both statements, including the nested
  `EXISTS` subquery inside `isDirectorOf`.
- **`pairDrain` no longer leaves interim orphan state.** The inner
  loop in `crm/src/context-compressor.ts` that matches tool results to
  a drained assistant message now scans the full conversation, not
  just the drainable zone. Session repair still runs after as
  defense-in-depth.
- **p95 latency semantics cleaned up.** `crm/src/tool-metrics.ts` now
  uses a named `percentile()` helper with nearest-rank semantics and
  explicit documentation. Behavior unchanged for NŌēź20 (correct
  tail); for small N the result degrades to max as before, which is
  the correct conservative choice when there's no data to distinguish.
- **Inference context limit raised to 100k.** `INFERENCE_CONTEXT_LIMIT`
  default in `crm/src/inference-adapter.ts` was 60k ŌĆö compression
  triggered at 48k for models (GLM-5, Qwen3.5+) that support 128k+.
  Default is now 100k with 20k headroom for completions and tool-call
  JSON. Override via env var for smaller models.

### Timezone sweep (Mexico City correctness)

- `crm/src/cross-intelligence.ts` quarter math now uses `getMxYear()`
  and `getMxDateStr()` helpers instead of `new Date().getFullYear()` /
  `.getMonth()`. Fixes Dec 31 MX ŌåÆ Jan 1 UTC off-by-one in vertical
  trend quarters.
- `crm/src/overnight-engine.ts` `fecha_expiracion` for calendar-driven
  insights now uses `getMxDateStr()` instead of
  `new Date().toISOString()`. No more "insight expired 6 hours early".
- `crm/src/warmth-scheduler.ts` and `crm/src/tools/relaciones.ts`
  `warmth_updated` timestamps now use `datetime('now','-6 hours')`
  SQL expression ŌĆö matches Mexico City time in briefings without
  mixing formats with the rest of the schema.
- `crm/src/proposal-drafter.ts` `fecha_creacion` and
  `fecha_ultima_actividad` for borradores use the same MX-SQL pattern.
- `crm/src/budget.ts` monthly window now honors Mexico City month
  boundaries: `datetime('now','-6 hours','start of month','+6 hours')`.
  Hourly and daily windows are UTC-safe because MX is a whole-hour
  offset and daily is relative; only monthly needed the fix.
- `crm/src/tools/perfil.ts` `fecha_actualizacion` on perfil writes
  uses `datetime('now','-6 hours')`.
- `crm/src/dashboard/api.ts` `getCuota` and `getDescarga` replaced
  `new Date().getFullYear()` with `getMxYear()`.
- New helper: `getMxDateTimeStr()` in `crm/src/tools/helpers.ts` for
  JS code paths that need an MX datetime string matching SQLite format.

### Performance ŌĆö hot path

- **Tool eviction is now fully async.** `crm/src/tool-eviction.ts`
  ported to `fs/promises`. `maybeEvict` is now async and awaited at
  its single call site in `inference-adapter.ts:761`. Probabilistic
  in-line cleanup removed ŌĆö replaced with a 10-minute interval
  scheduled from `bootstrap.ts` via the new `startEvictionCleanup()`.
  The TOC is now built from the full content (not the preview slice)
  so it reflects the whole document.
- **Briefing enrichment runs in parallel.** `crm/src/tools/briefing.ts`
  extracted `fetchClima` and `fetchFeriados` into separate functions
  and wraps them in `Promise.all`. Worst-case cold-cache briefing
  latency drops from ~10s to ~5s.
- **`consultar_cuentas` no longer N+1s.** `crm/src/tools/consulta.ts`
  replaced 3 correlated COUNT subqueries per row with 2 CTE scans
  using `SUM(CASE WHEN ...)`. For 20 accounts: 60 queries ŌåÆ 2 scans.
- **`ToolContext` is now cached per persona.** New
  `invalidateToolContextCache()` in `tools/index.ts` ŌĆö hierarchy
  lookups (`getTeamIds`, `getFullTeamIds`) no longer re-run on every
  tool-filtering pass. 10-minute soft TTL as safety net.
- **RAG query embeddings are LRU-cached.** `crm/src/doc-sync.ts`
  `embedQueryCached` wraps `embedText` with a 64-entry, 5-minute-TTL
  cache. Same query in the same round no longer hits the API twice.
- **New composite index.** `crm/src/schema.ts` adds
  `idx_propuesta_etapa_fecha ON propuesta(etapa, fecha_ultima_actividad DESC)`
  ŌĆö covers the most common pipeline query shape.
- **Overnight inventory N+1 removed.** `crm/src/overnight-engine.ts`
  pulls the candidate-accounts query outside the event loop instead
  of re-running it per event (10-50├Ś fewer queries on busy nights).
- **Cost ledger attributes to the actual provider.** `buildResult` in
  `inference-adapter.ts` now resolves provider name ŌåÆ model before
  calling `recordCost`. When the fallback provider handles a request,
  the cost row correctly reflects the fallback model, not the primary.

### Resilience

- **All Google Workspace Drive/Docs/Sheets calls now have explicit
  timeouts.** `crm/src/workspace/google/files.ts` adds `withTimeout`
  helper (15s reads, 30s writes) wrapping `drive.files.list`, `get`,
  `export`, `create`, `permissions.create`, and `update`. A partial
  Google outage can no longer hang inference indefinitely.
- **Hindsight circuit-breaker-open paths now log explicitly.**
  `crm/src/memory/hindsight-backend.ts` ŌĆö every "skipped because
  breaker open" and every recorded failure is surfaced via
  `logger.warn` with operation and bank. No more silent memory loss.
- **Sentiment classifier elevates after 5 consecutive failures.**
  `crm/src/sentiment.ts` tracks `consecutiveFailures` and logs ERROR
  (not WARN) once the threshold is crossed, making silent classifier
  breakage visible in dashboards.
- **Scheduler tasks can no longer double-fire.** `crm/src/scheduler.ts`
  checks for existing pending task files in the IPC directory before
  writing a new one. Prevents overlapping warmth/overnight runs on
  scheduler restart and the SQLITE_BUSY cascade that caused.
- **Approval IPC notifications timeout per recipient.**
  `crm/src/ipc-handlers.ts` wraps `deps.sendMessage` in a 5-second
  `Promise.race` inside a `Promise.all` fan-out. A stuck WhatsApp
  connection no longer blocks the whole broadcast; failures are
  collected and logged.
- **Doc-sync retries documents with `chunk_count = 0`.**
  `crm/src/doc-sync.ts` now checks `chunk_count` in the existence
  guard and, if a prior partial failure left a zero-chunk doc row,
  deletes it and re-embeds instead of silently skipping forever.
- **Embedding fallback is now surfaced.** `crm/src/embedding.ts`
  exports `isEmbeddingDegraded()` reflecting the most recent batch
  mode, so downstream tools can warn users when RAG quality is
  temporarily reduced to trigram fallback.

### Security

- **Spanish-aware homoglyph detection.** `crm/src/injection-guard.ts`
  expanded the `HOMOGLYPHS` map from Cyrillic-only to include Greek
  lookalikes, Latin Extended-A macron/caron variants, and
  Mathematical Alphanumeric bold italic letters. Regular Spanish
  diacritics (`├Ī ├® ├Ł ├│ ├║ ├▒ ├╝`) are _not_ in the map because they are
  legitimate characters.
- **Role scope enforced on `persona_nombre` resolution.**
  `crm/src/tools/analytics.ts` now has `isInScope()` and
  `resolveNameInScope()` helpers used by `cuotaScopeFilter`,
  `activityScopeFilter`, and `proposalScopeFilter`. A director or
  manager can no longer pass an arbitrary name and bypass their
  team filter ŌĆö out-of-scope names fall through to the default
  role filter instead.
- **Defense-in-depth for SQL identifier injection.**
  `crm/src/tools/perfil.ts` now uses an explicit
  `PROFILE_FIELD_COLUMNS` allowlist map ŌĆö if a future refactor
  accidentally weakens the enum check, the map lookup still produces
  either a known column name or `undefined`.
- **Dashboard dev JWT secret is random per process.**
  `crm/src/dashboard/auth.ts` replaces the hardcoded
  `crm-dashboard-dev-secret` with `crypto.randomBytes(32).toString('hex')`
  generated at module load when in non-production.
- **Dashboard queries have hard pagination caps.**
  `crm/src/dashboard/api.ts` defines `DASHBOARD_ROW_LIMIT = 500` and
  applies it to `getCuota` and `getDescarga`. `getPipeline` already
  had `LIMIT 100`.
- **`jarvis.ts` uses structured logger.** Three `console.log` /
  `console.warn` calls replaced with `logger.debug` / `logger.warn`
  on a child logger, so jarvis-pull telemetry is searchable and
  structured instead of scattered stdout.

### Intelligence / ACI

- **`buscar_documentos` and `buscar_web` descriptions rewritten.**
  Both previously had near-identical "busca en..." openings. Each
  now explicitly teaches WHEN to use it, what data source it covers,
  and a "NO USAR PARA" section. `buscar_documentos` also teaches the
  unicode61 tokenizer behavior (remove_diacritics, enum formatting).
- **AE persona teaches proactive memory use.** `crm/groups/ae.md`
  adds a "Memoria de largo plazo" section with 5 concrete trigger
  patterns: after losing a proposal (`buscar_memoria` then
  `guardar_observacion`), after winning a hard close, before a new
  stakeholder meeting, on detecting repeated objections
  (`reflexionar_memoria`), and when the user teaches the agent
  something new.
- **Manager persona teaches proactive Jarvis escalation.**
  `crm/groups/manager.md` expanded the `jarvis_pull` description
  with three proactive triggers: 3+ propuestas perdidas with same
  reason, repeated strategic questions beyond CRM data, and large
  accounts stalled 3+ weeks.
- **Global.md `clima` contradiction fixed.** Previous wording banned
  all `clima` from business scope, contradicting the
  `consultar_clima` tool that exists for OOH campaign planning.
  Rule now explicitly distinguishes _personal_ clima from _campaign
  planning_ clima.

## False positives excluded

The following audit claims were verified and rejected:

- Doom-loop `detectTextStalled` "inverted logic" ŌĆö equivalent
  statement, the code is correct.
- `.env` file world-readable ŌĆö actually 600, gitignored, not
  tracked.
- SQL injection via `${table}` in `aprobaciones.ts` ŌĆö gated by
  explicit `entidadTipo` whitelist that short-circuits on
  mismatch.
- SQL injection via `${campo}` in `perfil.ts` (as CRITICAL) ŌĆö gated
  by `VALID_CAMPOS.includes` check; downgraded to defense-in-depth
  (still fixed via explicit map lookup).
- `ipc-handlers` "unhandled async import exceptions" ŌĆö imported
  functions are synchronous (`export function` not `export async
function`), try/catch catches everything correctly.
- `reflexionar_memoria` "missing description" ŌĆö already had a full
  "CU├üNDO USAR / NO USAR" description in `tools/index.ts`.
- `jarvis_pull` "not in UNTRUSTED_TOOLS" ŌĆö was already in the set
  at `injection-guard.ts:34`.
- `isUntrustedTool` "called per result instead of cached Set.has"
  ŌĆö already backed by a module-level `Set` with `.has()`.
- `recomputeWarmth` "rebuilds all relationships" ŌĆö already per
  `relacionId`.
